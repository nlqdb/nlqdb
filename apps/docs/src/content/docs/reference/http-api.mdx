## Endpoints (overview)

- `POST /v1/ask` — the canonical NL→SQL endpoint. See [`NlqClient.ask`](/reference/sdk/type-aliases/nlqclient/).
- `POST /v1/run` — raw-SQL escape hatch (`GLOBAL-015`). See [`NlqClient.runSql`](/reference/sdk/type-aliases/nlqclient/).
- `POST /v1/databases` — create a database from a plain-English brief. See [`NlqClient.createDatabase`](/reference/sdk/type-aliases/nlqclient/).
- `GET /v1/databases` — list databases. See [`NlqClient.listDatabases`](/reference/sdk/type-aliases/nlqclient/).
- `POST /v1/keys`, `GET /v1/keys`, `DELETE /v1/keys/:id` — API-key lifecycle.
- `POST /v1/chat/messages` — chat-style transcript endpoint.

Curl-shaped examples for `/v1/ask` (read + write + anonymous mode) live in
the [curl tutorial](/tutorials/curl/).

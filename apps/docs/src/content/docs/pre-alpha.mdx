## Two ways through

### 1. Join the waitlist — public path

Open [nlqdb.com](https://nlqdb.com), describe what you're building, click **Create the DB**.
You'll see a progress panel ("BIRD 31.8% / 65%, Spider: not yet measured") and a
**Join the waitlist** button. That's it — we email you when the gate lifts.

The marketing hero, the in-product chat, the SDK, the CLI, MCP hosts, and raw
`curl` all return the same `feature_gated` envelope today; every surface
renders it as a one-sentence message + the
[waitlist URL](https://nlqdb.com/#waitlist).

### 2. Pass an invite code — design-partner path

If we've sent you a code, send it as `X-Invite-Code` on every request. It works
for anonymous **and** signed-in callers, so you can smoke-test the API before
creating an account.

## What the gate response looks like

```jsonc
{
  "error": {
    "status": "feature_gated",
    "message": "nlqdb is pre-alpha — join the waitlist for early access.",
    "action": "Join the waitlist",
    "waitlist_url": "https://nlqdb.com/#waitlist",
    "gate": {
      "bird_accuracy": 0.318,
      "spider_accuracy": null,
      "bird_target": 0.65,
      "spider_target": 0.75,
      "measured_at": "2026-05-18T22:42:29.917Z"
    }
  }
}
```

`gate.spider_accuracy: null` means the Spider 2.0-lite loader hasn't shipped yet
— the gate is structurally closed until both lanes report a numeric accuracy.

## Why the gate exists

Shipping a public NL→SQL surface at 31.8% BIRD execution-accuracy on the free
chain would burn the "great on free LLMs" thesis on impressions we can't take
back. Gating now buys time to land the engine scaffolding (planner, validator,
plan cache, retrieval, trust UX) before strangers form an opinion. When both
lanes clear, this page goes away.

Track the eval lane in the
[quality-eval feature doc](https://github.com/nlqdb/nlqdb/tree/main/docs/features/quality-eval/FEATURE.md);
the live thresholds live in
[`apps/api/src/gate/eval-baseline.ts`](https://github.com/nlqdb/nlqdb/blob/main/apps/api/src/gate/eval-baseline.ts).

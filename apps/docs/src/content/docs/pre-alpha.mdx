<Aside type="caution" title="The API is closed today">
  `POST /v1/ask`, `POST /v1/run`, `POST /v1/databases`, and `POST /v1/chat/messages`
  return **HTTP 403** with `error.status: "feature_gated"` until our free LLM chain
  clears two thresholds on the [quality-eval](https://github.com/nlqdb/nlqdb/tree/main/docs/features/quality-eval)
  harness: BIRD ≥ 65% **AND** Spider ≥ 75% execution-accuracy. Live numbers ship in
  every 403 body and on the marketing form — see [What the gate response looks like](#what-the-gate-response-looks-like).
  Listing endpoints (`GET /v1/databases`, `GET /v1/keys`, `GET /v1/chat/messages`)
  stay open.
</Aside>

## Two ways through

### 1. Join the waitlist — public path

Open [nlqdb.com](https://nlqdb.com), describe what you're building, click **Create the DB**.
You'll see a progress panel with the live BIRD and Spider numbers, plus a
**Join the waitlist** button. That's it — we email you when the gate lifts.

The marketing hero, the in-product chat, the SDK, the CLI, MCP hosts, raw
`curl`, **and the `<nlq-data>` / `<nlq-action>` embeds** all return the same
`feature_gated` envelope today; every surface renders it as a one-sentence
message + the [waitlist URL](https://nlqdb.com/#waitlist). The browser
elements show the CTA inline in the rendered pane with the live BIRD /
Spider numbers, so a copy-pasted `<nlq-data>` becomes a working sign-up
prompt instead of a dead "Authentication required." (`SK-ELEM-014`).

### 2. Pass an invite code — design-partner path

If we've sent you a code, send it as `X-Invite-Code` on every request. It works
for anonymous **and** signed-in callers, so you can smoke-test the API before
creating an account.

<Tabs syncKey="surface">
  <TabItem label="curl">
    ```bash
    curl https://app.nlqdb.com/v1/ask \
      -H "Authorization: Bearer sk_live_..." \
      -H "X-Invite-Code: $NLQDB_INVITE_CODE" \
      -d '{"goal": "an orders tracker", "ask": "how many orders today"}'
    ```
  </TabItem>
  <TabItem label="CLI">
    ```bash
    export NLQDB_INVITE_CODE=...                       # set once
    nlq new "an orders tracker"                        # picks it up
    nlq ask "how many orders today"                    # picks it up

    # or per-call:
    nlq ask --invite-code="$CODE" "how many orders today"
    ```
  </TabItem>
  <TabItem label="SDK">
    ```ts
    import { createClient } from "@nlqdb/sdk";

    const nlq = createClient({
      apiKey: process.env.NLQDB_API_KEY!,
      inviteCode: process.env.NLQDB_INVITE_CODE,
    });
    ```
  </TabItem>
  <TabItem label="MCP host">
    Add `NLQDB_INVITE_CODE` to the per-server `env` map your host already
    supports (Claude Desktop, Cursor, VS Code Continue, Cline all accept this
    shape). `nlq mcp install` writes it for you when the env var is set; for a
    manual edit it looks like:

    ```jsonc
    {
      "mcpServers": {
        "nlqdb": {
          // command/args as written by `nlq mcp install` or your host's connector UI
          "env": { "NLQDB_INVITE_CODE": "..." }
        }
      }
    }
    ```
  </TabItem>
</Tabs>

## What the gate response looks like

Live numbers ship in every block; the snippet below shows the response shape
(values illustrative — the canonical lane state lives in
[`apps/api/src/gate/eval-baseline.ts`](https://github.com/nlqdb/nlqdb/blob/main/apps/api/src/gate/eval-baseline.ts),
updated weekly by the [quality-eval](https://github.com/nlqdb/nlqdb/tree/main/docs/features/quality-eval/FEATURE.md) cron).

```json
{
  "error": {
    "status": "feature_gated",
    "message": "nlqdb is pre-alpha — join the waitlist for early access.",
    "action": "Join the waitlist",
    "waitlist_url": "https://nlqdb.com/#waitlist",
    "gate": {
      "bird_accuracy": 0.318,
      "spider_accuracy": null,
      "bird_target": 0.65,
      "spider_target": 0.75,
      "measured_at": "2026-05-18T22:42:29.917Z"
    }
  }
}
```

`gate.spider_accuracy: null` means the Spider 2.0-lite loader hasn't shipped yet
— the gate is structurally closed until both lanes report a numeric accuracy.

## Why the gate exists

Shipping a public NL→SQL surface while the free chain is far below target would
burn the "great on free LLMs" thesis on impressions we can't take back. Gating
now buys time to land the engine scaffolding (planner, validator, plan cache,
retrieval, trust UX) before strangers form an opinion. When both lanes clear,
this page goes away.

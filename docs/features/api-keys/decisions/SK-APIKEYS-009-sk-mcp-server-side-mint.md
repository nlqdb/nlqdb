# SK-APIKEYS-009 — `sk_mcp_*` minted server-side at `POST /v1/oauth/mcp-callback`, never displayed

- **Decision:** The hosted MCP OAuth flow mints `sk_mcp_<host>_<device>_*` inside `POST /v1/oauth/mcp-callback` (session-gated) atomically with a one-shot KV code. The plaintext bearer rides inside the KV value for the 60 s TTL; `apps/mcp/`'s `defaultHandler` redeems the code at `/v1/oauth/mcp-callback/redeem` and feeds the bearer to `OAuthProvider.completeAuthorization` as the grant's `props`. The user-agent never sees the plaintext; only the OAuth access token is in the user-facing redirect.
- **Core value:** Bullet-proof, Effortless UX, Seamless auth
- **Why:** [`SK-APIKEYS-007`](SK-APIKEYS-007-mint-via-post-keys.md) mints via `POST /v1/keys` only when the user can copy the plaintext (`sk_live_` dashboard mint) or write it to a host config (`nlq mcp install`). The hosted-MCP "paste a URL" flow has neither affordance — the agent has no terminal, the user has no copy moment. Minting inside the OAuth callback puts the bearer where the DO needs it (in `props`) without ever crossing a user-controlled surface; `SK-MCP-013`'s one-shot KV code is the only place the plaintext lives outside the bound DO.
- **Consequence in code:** `apps/api/src/oauth-mcp-bridge.ts`'s `mintBridgeCode` calls `mintSkMcpKey` + `hashKey` and stores both inside the KV-encoded `BridgeStoredCode`. `apps/api/src/index.ts` mounts the cookie-gated `POST /v1/oauth/mcp-callback` and the code-gated `POST /v1/oauth/mcp-callback/redeem`. The plaintext never lands in D1 (only the HMAC hash does, same as every other mint). PRs that surface the plaintext anywhere — span attribute, log line, response body outside this flow — fail review.
- **Alternatives rejected:**
  - Mint at redemption (after code check) — adds a hop; consent screen stalls visibly while the user-agent waits for OAuth to settle.
  - Return the plaintext to the consent screen → echo through the OAuth redirect — leaks the bearer into URL logs / browser history.
  - Mint via `POST /v1/keys` from the consent screen — same UX surface as the dashboard mint, but [`SK-APIKEYS-007`](SK-APIKEYS-007-mint-via-post-keys.md) requires `nlq mcp install` to write the result to a host config; the hosted flow has no host config to write to.

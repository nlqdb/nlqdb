# SK-SOLVE-001 — Every solve page uses a natural-language search query as `<h1>`, not a fabricated "verbatim" quote

- **Decision:** Each `/solve/<slug>` page renders the buyer's natural-language search query as `<h1>` — the question the persona types into Google / Perplexity / ChatGPT. Pages must NOT fabricate "verbatim" pain quotes attributed to anonymous users; the [§3.1 plan](../../../research/automated-icp-validation-plan.md) calls for verbatim cluster quotes, but until the ICP-mining cluster pipeline ([`icp-mining/FEATURE.md`](../../icp-mining/FEATURE.md)) emits its first evidence file (Mon 2026-05-26), an honest paraphrased search-intent question is the only ethically defensible shape. Each entry must also ship ≥3 `howNlqdbAnswers` bullets and ≥3 FAQs — the LLM-citation surface that earns the impression in the first place.
- **Core value:** Honest latency, Bullet-proof
- **Why:** AEO crawlers (Perplexity, ChatGPT, Claude) lift FAQ Q&A pairs verbatim into "direct answer" boxes and demote pages whose "quotes" are unverifiable. A paraphrased search query is more honest, more matchable against the actual queries LLMs route, and survives the audit a verbatim attribution would not. The 3-bullet / 3-FAQ floor is what the AEO citation surface needs — fewer and the page reads as a stub; the LLM passes it over for a better-detailed answer elsewhere.
- **Consequence in code:** `apps/web/src/data/solve.ts`'s `SolveEntry.searchTitle: string` is the `<h1>`. The shape lives in TypeScript so a missing or empty value fails type-check at build time. `solve.test.ts` pins `≥3 howNlqdbAnswers`, `≥3 faqs`, FAQ-answer word-count ≤80 (the LLM lift-verbatim sweet spot), and `searchTitle` length ≤90 chars (LLM citation-panel width).
- **Alternatives rejected:**
  - **Fake verbatim quotes** — the §3.1 plan asks for verbatim quotes from clustered evidence; until that evidence exists, fabricating quotes would violate `CLAUDE.md P1` (don't invent evidence-based content). Once the cluster file lands, follow-up entries amend `searchTitle` to the actual cluster label — that's a future SK, not this one.
  - **Skip the page until cluster data exists** — defers the acquisition surface ≥1 week and contradicts the §5.1 calendar's W2 "first 5 solve pages" target. The honest paraphrase ships now; the verbatim upgrade ships when the cluster file lands.
  - **Pull h1 from a free-text marketing field** — invites copy that drifts from the search query the user actually typed; the `searchTitle` field name pins the discipline.

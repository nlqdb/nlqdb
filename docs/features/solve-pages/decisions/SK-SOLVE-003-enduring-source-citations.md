# SK-SOLVE-003 — Every solve page cites ≥2 enduring discussion-hub URLs, never single-thread URLs

- **Decision:** Every `/solve/<slug>` page renders a `sources` block with ≥2 citations to **enduring** discussion-hub URLs (HN Algolia search results, subreddit URLs, GitHub label pages, official pricing pages). Single-thread URLs (`/r/X/comments/<id>`, `news.ycombinator.com/item?id=<n>`) are **prohibited** — they rot when the post is deleted, locked, or buried. Each entry must include a `label` describing what the reader will see; the page renders the block under "Where this pain shows up in public" so the reader can verify the theme without trusting our framing.
- **Core value:** Bullet-proof, Honest latency
- **Why:** The §3.1 plan's "credit complainer, don't outrank" pattern uses `<link rel="canonical">` pointing at a specific post. That pattern fails the moment the post 404s, gets renamed, or is privatised — and a 404 in our cited evidence is worse than no citation at all. Search-result URLs and subreddit URLs survive content churn because they're rendered on demand from a live index; the theme stays observable even as individual threads come and go. This is the same evidence-trail discipline `docs/research-receipts.md` enforces for the cited-research footer on the marketing site.
- **Consequence in code:** `apps/web/src/data/solve.ts`'s `SolveEntry.sources: SolveSource[]` is required, `≥2` entries enforced by `solve.test.ts`, each entry shape-checked: `url` starts with `https://`, `label` non-empty. The page template renders the block via `<a rel="nofollow noopener noreferrer" target="_blank">` to (a) avoid passing PageRank to the cited hub (most discussion hubs are already higher-PR than us), (b) protect against window-object leakage on cross-origin nav.
- **Alternatives rejected:**
  - **Cite specific posts (per §3.1 plan literal wording)** — link rot is the failure mode; once the post 404s, the citation surface lies. Reader sees "verify the theme is real" → clicks → 404 → trust loss.
  - **Allow `sources: []`** — page ships without evidence trail, indistinguishable from one-sided marketing. The plan calls for citations; the test pins ≥2.
  - **Cite our own ICP-mining cluster file** — the cluster file is internal evidence; the public-discussion hubs are external verification. Both are useful; this rule pins the external layer.

# SK-MCP-010 — Implementation slicing: keys → stdio → hosted → install

- **Decision:** Four ordered slices, each independently reviewable. Slice 3 is sub-sliced (3a → 3b → 3c) so the hosted Worker ships end-to-end before OAuth and rate-limit hardening land. **Slice 1 (shipped):** `sk_live_*` + `sk_mcp_*` mint / lookup in `apps/api/src/{api-keys.ts,principal.ts}` + canonical `POST /v1/keys` (`SK-APIKEYS-007`). **Slice 2 (shipped):** `packages/mcp/` local-stdio package. **Slice 3a (shipped):** `apps/mcp/` Cloudflare Worker (`nlqdb-mcp-server`) on `mcp.nlqdb.com` — MCP Streamable-HTTP at `/mcp`, stateless per-request `McpServer`, bearer-prefix gate forwarding to `apps/api/` via `@nlqdb/sdk`. **Slice 3b (shipped):** `workers-oauth-provider` + `McpAgent` Durable Object sessions per `SK-MCP-011..014` — dynamic client registration, single `mcp` scope, cross-Worker callback bridge minting `sk_mcp_*` server-side, *plus* the 1 s `McpAgent` revalidation cache for `SK-MCP-009`'s revocation half (originally scoped for 3c, pulled forward because the DO lifecycle was the natural home). **Slice 3c (shipped):** the remaining `SK-MCP-009` work — per-bucket rate-limit (`apps/api/src/principal.ts::rateLimitBucketKey` keys sk_* by `rl:${api_keys.id}`; migration 0014 renames `rate_limit_buckets.user_id` → `bucket_key`) + auth-failure observability (`nlqdb.mcp.http.request` span wraps non-health requests; `nlqdb.mcp.auth.failures.total{error_code, status}` counter fires from `OAuthProvider`'s `onError`). **Slice 4 (open, `cli/FEATURE.md`):** `nlq mcp install` host detection (Go) — consumes the `SK-MCP-008` registry.
- **Core value:** Simple, Honest latency, Goal-first
- **Why:** Shipping `packages/mcp/` before keys is intentional: tool contracts and dispatch are stable regardless of auth backend. Slice ordering is enforced by review, not code gates; the FEATURE.md is the source of truth.
- **Consequence in code:** `auth_required` responses follow `SK-MCP-006`'s shape; the action text in `packages/mcp/src/tools.ts` points users at `https://app.nlqdb.com/keys`. `apps/mcp/` holds OAuth state (KV) + per-session DO state but no D1 — `apps/api/` stays the auth-of-record. Reviewers reject sk-key minting logic inside `packages/mcp/` or `apps/mcp/`.
- **Alternatives rejected:**
  - Bundle slice 1 + slice 2 — doubles PR size; couples auth-backend work with package wiring.
  - Hosted-only (skip stdio) — cuts off the offline / air-gapped segment (`SK-MCP-001`).

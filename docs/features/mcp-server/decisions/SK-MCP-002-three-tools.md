# SK-MCP-002 — A small fixed verb set, no `nlqdb_create_database`: `nlqdb_query`, `nlqdb_list_databases`, `nlqdb_describe` (+ additive `nlqdb_remember`, `nlqdb_connect_database`)

- **Decision:** The MCP server exposes a small, fixed verb set: `nlqdb_query(q, db?)`, `nlqdb_list_databases()`, `nlqdb_describe(db)`, and — additively — `nlqdb_remember(db, kind, payload, …)` (E-02 / SK-PIVOT-008, the memory **write** verb, for the agent-memory wedge) and `nlqdb_connect_database(engine, connection_url, name?)` (the BYO-connect verb; its decision lives in `docs/features/byo-connect/FEATURE.md` under `GLOBAL-003` parity). There is **no public `nlqdb_create_database` tool** — `nlqdb_query` materializes the DB on first reference (per the goal-first inversion in `docs/architecture.md §0.1`). `db` is optional on `nlqdb_query`: omitting it lets `/v1/ask` auto-target the caller's only DB, create one from the goal when none exists, or return `ambiguous_db` with candidates on a multi-DB key — keeping the agent's prompt goal-first. The contract is **stable + additive**: existing tools are never renamed or removed (SK-PIVOT-006); a genuinely new *verb* (a distinct intent, not a second way to do an existing one) may be added with the GLOBAL-017 justification below.
- **Core value:** Simple, Goal-first, Effortless UX
- **Why:** Two tools (`create`, then `query`) doubles the prompt the agent has to learn and creates an "agent forgot to call create" failure mode. One tool that does the right thing is the goal-first design applied to MCP — the agent never had a goal that was "create a database". Implicit creation is also what makes the persona walkthrough (P2 / Jordan) work: the system prompt has one tool, not two. `nlqdb_remember` is a **distinct verb**, not a second way to query — a structured memory *write* (deterministic, no LLM) that `nlqdb_query` then reads/aggregates over; under GLOBAL-017 ("one way to do each thing") read and write are different things, so it adds a way to do a *new* thing, not a duplicate path.
- **Consequence in code:** `packages/mcp/src/tools.ts` + `server.ts` register the handlers; `nlqdb_query` POSTs to `/v1/ask` (typed-plan create path on first reference, `docs/architecture.md §3.6.2`), `nlqdb_remember` POSTs to `/v1/memory/remember` (server-built parameterised INSERT, SK-PIVOT-008). PRs adding further tools require explicit GLOBAL-017 justification (a new verb, not a duplicate) — renames/removals are out.
- **Alternatives rejected:**
  - Expose `nlqdb_create_database` as a power-user tool — dilutes the agent's prompt, contradicts §0.1 inversion.
  - One mega-tool that takes an `op` parameter — harder for the host LLM to plan with, no real simplification.
  - Fold memory writes into `nlqdb_query` (a `write:` arg) — overloads the read verb with a structured-write shape the LLM would have to learn anyway; a named `nlqdb_remember` is clearer to the host and keeps the read path's contract untouched.

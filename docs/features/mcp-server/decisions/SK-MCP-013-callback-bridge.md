# SK-MCP-013 — Cross-Worker callback bridge for OAuth consent

- **Decision:** `apps/mcp/`'s `defaultHandler` redirects `/authorize` to `app.nlqdb.com/oauth/mcp-authorize` (web app, Better Auth cookie scope). The consent page POSTs to `app.nlqdb.com/v1/oauth/mcp-callback` (session-gated via `requireSession`) which mints a one-shot 16-byte hex code AND the bound `sk_mcp_<host>_<device>_*` plaintext atomically; both live in KV with a 60 s TTL keyed by the code. The page redirects the browser to `mcp.nlqdb.com/oauth/mcp-bridge-callback?code=…`. `apps/mcp/` redeems the code at `app.nlqdb.com/v1/oauth/mcp-callback/redeem` (code-gated — the code itself is the auth proof, delete-on-read) and feeds the result to `OAuthProvider.completeAuthorization` as the grant's `props`.
- **Core value:** Bullet-proof, Simple, Seamless auth
- **Why:** Two Workers share no cookie scope (`mcp.nlqdb.com` ≠ `app.nlqdb.com`). Without the bridge, the consent page would have to live on `mcp.nlqdb.com` and somehow honor Better Auth — duplicating auth code or proxying sessions, both fragile. The one-shot code is a 128-bit random + 60 s TTL + delete-on-read; it's cryptographically equivalent to a session cookie for the brief window between consent and grant. `Idempotency-Key` (`GLOBAL-005`) on the callback path keys on `(user_id, idempotency_key)` per `SK-IDEMP-002` so a network retry returns the same code instead of double-minting.
- **Consequence in code:** `apps/api/src/oauth-mcp-bridge.ts` owns `parseBridgeBody`, `mintBridgeCode`, `redeemBridgeCode`, and the two handlers (`handleMcpCallback` cookie-gated, `handleMcpCallbackRedeem` code-gated). `apps/mcp/src/oauth-bridge.ts`'s `bridgeHandler` implements the redirects + redemption. The consent screen is `apps/web/src/pages/oauth/mcp-authorize.astro` (served on `app.nlqdb.com`, the Better Auth cookie scope) with its security-sensitive pure logic in `apps/web/src/lib/mcp-consent.ts`: it decodes the (unverified, re-checked on the `mcp.nlqdb.com` side) `flow` payload for the mint body, derives `mcp_host` from `client_name`, and **allowlists the `callback` to exactly the bridge-callback URL on the trusted MCP origin** — the one-shot code redeems to an `sk_mcp_*` bearer, so a `callback` to any other origin would be account-scoped credential exfiltration. A new OTel span `nlqdb.oauth.mcp_callback` (and `nlqdb.oauth.mcp_callback_redeem`) covers the mint + redeem hop per `GLOBAL-014`. The plaintext bearer lives in KV only for the 60 s TTL; the HMAC hash is the only persistent form in D1.
- **Alternatives rejected:**
  - Subdomain `auth.nlqdb.com` cookie shared by both Workers — Better Auth's cookie cache is scoped to one origin; cross-origin sharing breaks the 5-minute cache invariant (`SK-AUTH-007`).
  - Pass the session JWT verbatim through a redirect — JWT in URL gets logged, breadcrumbed, archived; the one-shot code is shorter-lived and never replayed.
  - Mint the `sk_mcp_*` only at redemption time (after code redemption) — adds a hop and lets a slow redemption stall the OAuth flow visibly; minting both atomically at consent is the lower-latency design.

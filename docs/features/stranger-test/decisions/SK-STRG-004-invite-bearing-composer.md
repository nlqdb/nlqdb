# SK-STRG-004 — Invite-bearing composer (`stranger-test-invited.sh`) mints one SK-GATE-007 code per run and drives FLOW-001/002/003 through the browser-side capture + header-forward path

- **Decision:** `scripts/stranger-test-invited.sh` composes the two existing walkers in one bash file: (1) invokes `scripts/flow-004-walk.sh` with `FLOW_004_INVITE_OUT=<sidecar>` to mint a fresh SK-GATE-007 invite via the mail.tm anonymous-bearer flow and write the raw code to a mode-600 sidecar file under `tools/stranger-test/results/.invite-<utc>.txt`; (2) reads-and-immediately-deletes the sidecar, validates the code against `/^[A-Za-z0-9_-]{16,128}$/`, then exports it as `NLQDB_INVITE_CODE` and invokes `bash scripts/stranger-test.sh --out <walk-invited-<utc>.json>` — the runner forwards `--invite-code` (or the env) into every `walkFlowNNN` call. The walkers prepend `?invite=<c>` to the navigation URL via the shared `withInviteParam(path, code)` helper, then assert `localStorage["nlqdb_invite"] === code AND ?invite= stripped` via `assertInviteCaptured(page, stepNum, code)` — this is the first regression detector for `apps/web/src/lib/invite.ts::captureInviteFromUrl`. The submit step (FLOW-001 step 5, FLOW-002 step 9, FLOW-003 step 8) flips its failure detail to call out the SK-GATE-007 regression signature (`feature_gated WITH invite`). Trap-based cleanup wipes the sidecar AND scrubs the in-process variable on every exit path (`EXIT INT TERM`); `unset FLOW_004_INVITE_OUT` before forking the child runner so the code can't be re-read indirectly. Per-run cost: ONE invite (200/week cap), ONE Resend send (3k/mo).

- **Core value:** Bullet-proof, Goal-first, Simple

- **Why:** [`SK-STRG-002`](../FEATURE.md) proves the gate honours `X-Invite-Code` over HTTP; [`SK-STRG-001`](../FEATURE.md) proves the static surfaces render. Neither proves the path a real stranger takes: land on `/?invite=<c>`, browser stores it, web app forwards it on `/v1/ask`. The first live run of this composer caught a real regression — `/solve/<slug>` and `/vs/<slug>` did not call `captureInviteFromUrl()`, so a stranger landing on a launch URL pointed at `/solve/cheap-internal-dashboard?invite=…` lost the code at the first `location.assign`. Fix landed alongside this SK in the same PR. The walker also recorded the **first HTTP 200 ever on `/v1/ask`** through the browser path (FLOW-001 step 5, ttfvMs ≈ 4 s) — surfaces a separate trace-toggle regression that the gate-403 was masking. Composition (not duplication) keeps the mail.tm flow in one file; the sidecar-file seam is the minimal API both walkers can share without coupling.

- **Consequence in code:** `withInviteParam(path, code)` and `assertInviteCaptured(page, step, code)` live in `tools/stranger-test/src/browser.ts` so any future flow walker plugs in via two lines. `WalkResult.inviteBearing` is the JSON discriminant the §1.2 KPI dashboard reads to split TTFV by mode (gate-403 vs invite-200 are different distributions). Invite codes never appear in stdout or JSON — the `redact()` helper (first-4 + last-4, refuses < 12 chars) is shared with `scripts/flow-004-walk.sh`. Acquisition surfaces added later (e.g. `/blog/<slug>`, `/launch/<slug>`) must also call `captureInviteFromUrl()` on load, or the invite-bearing walker will fail step 10 against them — this is the regression-detection value, not a bug.

- **Alternatives rejected:**
  - **Single TS workspace that re-implements the mail.tm flow.** Duplicates [`SK-STRG-002`](../FEATURE.md); two regression surfaces.
  - **Pass invite code via process argv (`--invite-code <c>`) without env.** Appears in `/proc/<pid>/cmdline` for any sibling on the runner; env-only scoping is at least one privilege boundary.
  - **Test-only `/v1/waitlist?reveal=1` returning the invite in response.** Backdoor that survives prod by accident — already rejected under [`SK-STRG-002`](../FEATURE.md).
  - **Wire the cron to run this daily instead of `flow-004-walk.sh`.** Doubles invite consumption (1 → 2/day) without proving anything FLOW-004 doesn't already prove cheaper. Gate until a real `?invite=` regression lands or BIRD/Spider clear — when the curl walker becomes `inconclusive`, this is its replacement.

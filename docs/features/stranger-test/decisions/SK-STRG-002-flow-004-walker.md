# SK-STRG-002 — FLOW-004 walker uses a throwaway mail.tm inbox via curl+jq, no browser, no shared inbox; verifies waitlist signup → welcome email → first-value ask

- **Decision:** `scripts/flow-004-walk.sh` is a single bash script that walks FLOW-004 end-to-end with zero pre-provisioned credentials. Per invocation it (1) `GET api.mail.tm/domains` to pick an active public domain, (2) `POST /accounts` to mint a throwaway inbox + `POST /token` for a bearer JWT, (3) `POST $NLQDB_BASE_URL/v1/waitlist` with the throwaway address (`source: "flow-004-walker"`, `persona: "solo-builder"`), (4) polls `GET api.mail.tm/messages` every `FLOW_004_POLL_INTERVAL_S` seconds (default 10) up to `FLOW_004_TIMEOUT_S` (default 300) for the welcome/confirmation email whose `from.address` matches `/nlqdb/i`, (5) `POST /v1/ask` with `Authorization: Bearer anon_<uuid>` and verifies a real first-value HTTP 200. Decision matrix: `passed` = waitlist signup + email received + `/v1/ask` HTTP 200; `failed` = email never arrived or `/v1/ask` non-200; `partial` = HTTP 200 but degraded first-value (see [`SK-STRG-006`](../FEATURE.md#sk-strg-006--flow-004-walker-grades-first-value-quality-on-the-http-200-not-just-reachability)). The trap-registered cleanup runs `DELETE /accounts/{id}` on every exit path, including a Ctrl-C between account-create and token-fetch (the trap re-mints the JWT from the saved password if needed). Outcome JSON shape (uniform across success/failure): `{utc, flow, base_url, mail_tm_domain, state, email_latency_s, total_wall_s, ask_status, ask_error_status, notes}` — [`SK-STRG-006`](../FEATURE.md#sk-strg-006--flow-004-walker-grades-first-value-quality-on-the-http-200-not-just-reachability) appends the first-value-quality fields (`first_value_kind`, `first_value_quality`, `result_status`, `row_count`, `table_count`, `answer_confidence`, `answer_model`, `sql_is_select`). Per-run cost: one waitlist send (Resend, 3k/mo free tier).

- **Core value:** Bullet-proof, Simple, Goal-first

- **Why:** Before this script, FLOW-004 had been "shipped not verified" since 2026-05-21 — the waitlist signup → welcome-email → first-query path has a Resend-template / waitlist-handler regression surface no one could exercise without an inbox. mail.tm exposes a free anonymous bearer-token REST API ([`docs.mail.tm`](https://docs.mail.tm/)) that mints disposable inboxes at 8 QPS without signup. First live walk recorded `HTTP 200` in 18s. Bash+curl matches `verify-flows.sh`'s precedent. The walk is a regression detector, not a static "API up" check: it proves a real stranger can sign up, receive the email, and get a first-value answer on the deployed surface.

- **Consequence in code:** The `redact()` helper (first-4 + last-4 only, refuses < 12 chars) is the contract for any agent-facing log touching JWTs or throwaway mail.tm addresses. The `write_outcome` helper centralises the JSON shape so every failure path emits a uniform record — no triage-blind exit. Output JSON lives under `tools/stranger-test/results/` so a future R2-archive cron sweeps both walkers with one glob. The [`SK-STRG-008`](./SK-STRG-008-flow-004-seed-quality-probe.md) seed-quality probe composes this walker to set up, then issues `create` asks across many goals to measure `seeded_ok_ratio`.

- **Alternatives rejected:**
  - **Resend webhook capture.** Public ingest endpoint adds Worker-side state. mail.tm needs zero infra changes.
  - **Dedicated nlqdb-owned test inbox.** Adds DNS / IMAP / OAuth surface; every run lands in the same mailbox, creating race conditions and stale-message reuse.
  - **Test-only `/v1/waitlist?reveal=1` returning internal state in the response body.** Creates a backdoor that survives prod by accident.
  - **Mock-IdP injection via Cloudflare API.** Skips Resend + the email-delivery path — exactly the regression the walker exists to catch.
  - **TS workspace under `tools/flow-004-walker/`.** Heavier than `scripts/verify-flows.sh`'s precedent; the agent can grep the whole walk in one file.

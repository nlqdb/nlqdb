# SK-STRG-005 — FLOW-005 MCP discovery + auth-wall walker; HTTP-only, no credentials, asserts RFC 9728 challenge metadata matches scoped discovery

- **Decision:** `scripts/flow-005-walk.sh` is a single bash script that exercises the FLOW-005 surface every MCP client (Claude Desktop, Cursor, Cline, ChatGPT desktop, `@modelcontextprotocol/inspector`) hits before it can prompt the user for an `sk_mcp_*` key, with zero pre-provisioned credentials. Per invocation it (1) `GET $MCP_URL/.well-known/oauth-protected-resource` — 200 + `resource` equals the canonical MCP URL (RFC 9728 root variant), (2) `GET $MCP_URL/.well-known/oauth-protected-resource/mcp` — 200 + `resource` equals `$MCP_URL/mcp` (RFC 9728 resource-scoped variant), (3) `GET $MCP_URL/.well-known/oauth-authorization-server` — 200 + `issuer` + `authorization_endpoint` + `token_endpoint` (RFC 8414), (4) `POST $MCP_URL/mcp` with the JSON-RPC `initialize` request and no `Authorization` header — must return 401 with `WWW-Authenticate: Bearer realm=*, resource_metadata=*, error="invalid_token"` AND the `resource_metadata` URL must equal the scoped discovery URL from step 2, (5) `POST $MCP_URL/mcp` with `tools/list` and no auth — same 401-with-challenge shape (proves the wall isn't method-specific). Output JSON: `{utc, flow:"FLOW-005", base_url, state, total_wall_s, checks_passed, checks_failed, discovery_ok, auth_wall_ok, challenge_url_matches, notes}` under `tools/stranger-test/results/flow-005-<utc>.json` (gitignored). Pass = all 6 assertions green AND `discovery_ok` AND `auth_wall_ok` AND `challenge_url_matches`; `failed discovery` / `failed auth wall` / `failed challenge URL` are the three triage states.

- **Core value:** Bullet-proof, Simple, Goal-first

- **Why:** FLOW-005 is the P2 (agent builder) inbound and one of the five canonical user flows per [`GLOBAL-032`](../../../decisions/GLOBAL-032-top-5-user-flows-canonical.md), yet it was the only canonical-five flow without a dedicated agent-runnable walker — `verify-flows.sh` covered the static OAuth discovery endpoints (steps 1+3) but never exercised the auth wall or asserted the RFC 9728 challenge URL matches discovery. A regression in any of (a) the scoped resource-metadata endpoint, (b) the WWW-Authenticate header presence, or (c) the `resource_metadata` URL pointing at the scoped discovery would silently break every MCP client without any test catching it — the inspector handshake would fail with an opaque "missing OAuth metadata" message and the agent would assume the server is down. The walker fails loudly on every one of those shapes; per-run cost is five HTTP requests, no credentials, no state, no side effects, ≤ 4 s wall-clock. Matches `verify-flows.sh` / `flow-004-walk.sh` shape so adding it to the daily cron is one step.

- **Consequence in code:** Adding the walker to the daily `acquisition-health.yml` cron means `flow-005-walk.sh` is re-run every 24 h (with `continue-on-error: true` per SK-STRG-003) and the outcome JSON joins the 90-day artifact. The walker has no extension seam — FLOW-005 walkthrough steps 6+ (authenticated `tools/list`, `create_database`, `ask`, `run`) still need an `sk_mcp_*` key and stay in the verification mirror for the credentialed walk. Future MCP server protocol bumps (e.g. `2025-06-18` MCP protocol version) only affect the `initialize` payload — the auth-wall shape (401 + WWW-Authenticate) is the MCP/OAuth contract that needs to stay invariant.

- **Alternatives rejected:**
  - **Extend `verify-flows.sh` FLOW-005 block instead of a new walker.** Mixes "static curl-observable subset" with "the dedicated per-flow walker"; the per-flow walker pattern (`flow-004-walk.sh` precedent) is the right shape because [`GLOBAL-032`](../../../decisions/GLOBAL-032-top-5-user-flows-canonical.md) demands one agent-runnable walker per canonical flow. `verify-flows.sh`'s FLOW-005 stays as the lightest possible static probe; this walker is the full discovery + auth-wall test.
  - **Use `@modelcontextprotocol/inspector` as the walker.** Interactive CLI — not agent-runnable without a TTY harness; takes ~30 s to handshake; doesn't probe the 401-without-credentials path because the inspector always tries to authenticate first.
  - **Spin up a real MCP client (Claude Desktop / Cursor) in CI.** GUI applications — impossible to run headlessly on a GH Actions runner without a desktop emulator. Even if possible, the credential surface widens dramatically.
  - **Send only `initialize` and not also `tools/list`.** Two probes prove the wall isn't method-specific. A 200 on `tools/list` without auth would be the actual security regression — proving 401 on both is the minimal cover.
  - **Assert the challenge URL pattern without comparing to step 2's body.** Misses the regression where the WWW-Authenticate `resource_metadata` URL points at a non-existent (or stale) discovery endpoint. The cross-check is the highest-value assertion of the walker.

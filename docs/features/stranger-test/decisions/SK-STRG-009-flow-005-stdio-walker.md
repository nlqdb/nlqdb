# SK-STRG-009 ŌĆö FLOW-005 local-stdio-transport walker: spawns the real `@nlqdb/mcp` binary and asserts the `initialize` + `tools/list` catalog an npm-fallback install discovers

- **Decision:** `scripts/flow-005-stdio-walk.sh` (a thin wrapper over `tools/stranger-test/src/flow-005-stdio.ts`) is the FLOW-005 walker for the **local-stdio transport** ŌĆö the npm-fallback install path from [`SK-MCP-001`](../../mcp-server/decisions/SK-MCP-001-two-transports.md) that a Claude Desktop / Cursor user runs when they paste a key into the host config instead of using the hosted connector URL. It spawns the real `packages/mcp/bin/nlqdb-mcp.mjs` binary via `StdioClientTransport` and drives a real MCP `initialize` + `tools/list` handshake over OS pipes ŌĆö no mocking, no network. `initialize` and `tools/list` are served from the in-memory tool registry, so the `stdio.ts` `NLQDB_API_KEY` prefix-gate is satisfied with a throwaway `sk_live_ŌĆ”` token that never authenticates (the walk stops before any tool *invocation*, which would). It asserts the exact catalog an MCP host discovers before it can call any tool: `nlqdb_query` (destructiveHint) + `nlqdb_list_databases` + `nlqdb_describe` (readOnlyHint) + `nlqdb_remember` (the additive E-02 memory-write verb), each with the right input-schema keys, and **no `create_database` / `ask` / `run` tool** (create is implicit via `nlqdb_query`, per [`SK-MCP-002`](../../mcp-server/decisions/SK-MCP-002-three-tools.md)). Output JSON `{utc, flow:"FLOW-005", transport:"stdio", state, total_wall_s, checks_passed, checks_failed, protocol_ok, catalog_ok, server_name, tools, notes}` to `tools/stranger-test/results/flow-005-stdio-<utc>.json` (gitignored). Pass = every check green AND `protocol_ok` AND `catalog_ok`; exit 0 pass / 1 contract-assertion fail / 2 harness (spawn / handshake) error. The pure `assessHandshake()` catalog assertion is unit-tested in `tools/stranger-test/test/flow-005-stdio.test.ts`.

- **Core value:** Bullet-proof, Simple, Goal-first

- **Why:** FLOW-005 (P2 agent builder) is one of the five canonical flows per [`GLOBAL-032`](../../../decisions/GLOBAL-032-top-5-user-flows-canonical.md), but [`SK-STRG-005`](./SK-STRG-005-flow-005-walker.md) only walks the *hosted* transport's discovery + auth-wall on `mcp.nlqdb.com` ŌĆö the *stdio* transport (`SK-MCP-001`'s second of two transports) had zero e2e coverage. An MCP host's first action on either transport is the `initialize` + `tools/list` handshake; if the tool catalog regresses (a renamed tool, a dropped trust-annotation hint, a changed input-schema key, an accidental public `create_database` tool that would violate `SK-MCP-002`), every npm-fallback install silently breaks with an opaque "tool not found" at use time and no test catches it. The credentialed hosted path stays OAuth-blocked (it needs a browser login to mint an `sk_mcp_*`), so the stdio handshake is the deepest FLOW-005 surface an agent can walk end-to-end with no pre-provisioned credentials ŌĆö and it is the surface that proves the catalog contract for *both* transports, since both register tools through the same `createServer()` in `@nlqdb/mcp`. Per-run cost: one short-lived child process, no network, ~0.3 s wall.

- **Consequence in code:** `tools/stranger-test/package.json` gains `@modelcontextprotocol/sdk` pinned to `packages/mcp`'s version (`1.29.0`) so the walker's MCP client never drifts from the server's protocol. The walker joins the daily [`acquisition-health.yml`](../../../../.github/workflows/acquisition-health.yml) cron as a `continue-on-error: true` step with a summary row + outcome block, satisfying the `GLOBAL-032` seven-day freshness rule by default. [`GLOBAL-032`](../../../decisions/GLOBAL-032-top-5-user-flows-canonical.md) ┬¦Consequence (4) lists FLOW-005 ŌåÆ both `flow-005-walk.sh` (hosted) and `flow-005-stdio-walk.sh` (stdio). Authenticated tool *invocation* (`nlqdb_query` against a real DB, `nlqdb_list_databases`, `nlqdb_describe` with a user-scoped key) still needs an `sk_mcp_*`/`sk_live_*` key and stays in the verification mirror for the credentialed walk. A future MCP protocol-version bump only changes the `initialize` payload the SDK negotiates; the catalog contract this walker pins is transport- and version-independent.

- **Alternatives rejected:**
  - **Extend `flow-005-walk.sh` to also spawn the stdio server.** That walker is HTTP-only (curl + jq) by design; spawning a node child + speaking JSON-RPC over stdio is a different runtime. A separate walker keeps each one single-responsibility and lets the cron report the two transports independently.
  - **Pass a real `sk_mcp_*`/`sk_live_*` key and invoke a tool end-to-end.** No such key is mintable without an OAuth browser login or D1 write access (keys are stateful ŌĆö `lookupSkKey` hits D1); the credentialed walk stays in the mirror until a CI secret or design-partner key exists. Faking it would violate the no-mock bar.
  - **Assert the catalog by importing `createServer()` directly and inspecting its registry.** Skips the real transport ŌĆö it would pass even if the published `bin` or the `StdioServerTransport` wiring regressed. Spawning the actual binary is what proves the npm-fallback install path works.
  - **Only assert tool names, not annotations + input schemas.** The trust-annotation hints (`destructiveHint` / `readOnlyHint`, `SK-MCP-002` / `GLOBAL-023`) and the input-schema keys are the part an MCP host renders and fills in; a name-only check would miss the regressions that actually break the agent UX.

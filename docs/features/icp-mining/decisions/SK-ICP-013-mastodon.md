# SK-ICP-013 — Mastodon (`mastodon.social` public hashtag timeline) as the 9th pain-signal source

- **Decision:** `runIcpScrape` adds a 9th `Promise.all` element calling `https://mastodon.social/api/v1/timelines/tag/<tag>?limit=25&local=false` for 5 P1/P2/P3/P6 hashtags (`postgres`, `database`, `sql`, `llm`, `rag`). Stored as `source: "mastodon"`, `id: "mast-<post.id>"`; the post's federated `url` is preserved verbatim (cross-instance posts keep their origin host so the evidence file cites the real author). `content` is HTML — `stripMastodonHtml` strips tags + decodes the 7 entities Mastodon's renderer emits before storage so the LLM scoring/clustering pass sees plain language. Posts older than the rolling 7-day window, `sensitive: true` posts (NSFW — evidence file is product-public), and posts missing `id` / `url` / `created_at` are dropped pre-write. A single 429 short-circuits the remaining queries (Mastodon documents 300 reads / 5 min per IP). Shared `BOT_USER_AGENT`, `FETCH_TIMEOUT_MS` (10s), per-source `.catch`. No auth, no env binding. Live probe 2026-06-04 from the agent VM: HTTP 200 + JSON array on every probed tag; `postgres` → 20 fresh posts in 24 h incl. *"Handling graphs with SQL/PGQ in PostgreSQL"* (P3 analyst signal); `llm` / `rag` / `agent` each return 20 fresh posts (P2 agent-builder signal). `x-ratelimit-remaining=294` after 6 probes (cron uses 5/week — three orders of magnitude inside the bar).

- **Core value:** Simple, Bullet-proof

- **Why:** Mastodon reaches a federated dev demographic that has lower overlap with Bluesky than HN does with Reddit. SK-ICP-012 covered the AT Protocol side of the post-2024 X exodus; SK-ICP-013 covers the ActivityPub side. `mastodon.social` is the largest public AP instance and federates posts from thousands of others, so a hashtag query returns the global timeline for that tag across the fediverse — strictly cheaper than probing each instance separately. The `/api/v1/timelines/tag/<tag>` endpoint is documented as `OAuth: Public` (https://docs.joinmastodon.org/methods/timelines/) — no auth, no app registration, no token rotation. `mastodon.social/robots.txt` explicitly disallows GPTBot but leaves `/api/v1/timelines/tag/*` open for any other user-agent; ours is `nlqdb-icp-bot` (matches the existing IH/GH/Bluesky bot identity). Rate limit: `docs.joinmastodon.org/api/rate-limits/` documents 300/5min for authenticated users; the unauthenticated quota is undocumented but the agent VM observed `X-RateLimit-Limit: 300, X-RateLimit-Remaining: 294` after 6 probes on 2026-06-04 — the code reads `X-RateLimit-Remaining` as the canonical budget sensor (the way GitHub Discussions reads `rateLimit.cost`). Cron uses 5 calls/week — three orders of magnitude inside the observed bar.

- **Consequence in code:** `apps/api/src/icp-scrape.ts` gains `MASTODON_QUERIES`, `MASTODON_TIMELINE_URL`, `MastodonPost` type, `stripMastodonHtml` helper (decodes `&amp;` last so a literal `&amp;lt;` in a how-to about HTML doesn't collapse to `<` mid-pass), `fetchMastodon`, and a 9th `Promise.all` element. Pre-write rejects: missing `id` / `url` / `created_at`; non-`http(s)` `url` (federation trust boundary — a malicious origin instance can't push `javascript:` into the evidence file); `sensitive: true` (NSFW); `visibility !== "public"` (`unlisted` posts opted out of bulk indexing); `created_at` older than 7 days; empty stripped text. OTel span `nlqdb.icp.fetch.mastodon` carries `nlqdb.icp.source` + `http.response.status_code` + `nlqdb.icp.items_returned` (count from the AppView) + `nlqdb.icp.items_stored` (count after the pre-write rejects — split so future log-based alerts can distinguish "source dead" from "source returns NSFW-only") + `nlqdb.icp.mastodon.rate_remaining` (set ONLY when the `x-ratelimit-remaining` header is present — a missing header is NOT recorded as `0` because `Number(null) === 0` would falsely signal "near exhausted"). LogSnag adds `MAST: <n>` after `BSKY: <n>`. `scripts/verify-flows.sh` FLOW-008 gains a `GET /api/v1/timelines/tag/postgres` probe (fatal severity; non-egress-blocked from the agent VM — confirmed 2026-06-04) asserting HTTP 200 + JSON-array body. `apps/api/test/icp-scrape.test.ts` gains 10 tests: HTML-stripped storage with `mast-<id>` prefix; URL+headers contract (`limit=25`, `local=false`, `User-Agent: nlqdb-icp-bot`, path-segment-encoded tag — never raw whitespace); 429 short-circuit on the first call; 7-day window drop for old posts; sensitive=true NSFW drop; 503 graceful (other sources still run); headers-absent `rate_remaining` omission (regression test for `Number(null)`); `visibility=unlisted` drop; non-http(s) `url` drop; `&amp;lt;` decode-order check.

- **Alternatives rejected:** Per-instance probes (federated nature makes per-instance brittle; `mastodon.social` AppView is the canonical hub); the `/api/v2/search` endpoint (requires OAuth for tag results); X/Twitter (no free post-2023 per impl plan §2.1 skip line); Lobste.rs (rejected 2026-05-25 per `robots.txt`); `programming.dev` Lemmy (rejected 2026-05-30 — `Content-Signal: ai-train=no` + explicit ClaudeBot/GPTBot/CCBot Disallow); leaving HTML in the stored `text` (LLM scoring pass would see `<p>` / `<a>` markup that doesn't carry any pain signal and inflates token counts); decoding `&amp;` first (would collapse literal `&amp;lt;` to `<` mid-pass — ambiguous text for the LLM); recording `Number(res.headers.get("x-ratelimit-remaining"))` unconditionally (returns `0` when the header is missing, falsely signaling exhaustion).

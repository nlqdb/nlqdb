# SK-ICP-010 — Drop the pain-word regex prefilter; LLM relevance floor is the only gate

- **Decision:** `runIcpScore` no longer regex-prefilters scraped items on a fixed pain-word list before scoring. Every scraped item is sent to the LLM scorer; `RELEVANCE_FLOOR = 5` on the best persona score is the sole relevance gate. Supersedes the prefilter clause of SK-ICP-002 (the rest of SK-ICP-002 — batching, floor, KV schema, `.catch` wrapping — stands). `IcpScoreResult.skipped` is now non-zero only when no LLM key is configured.
- **Core value:** Simple, Bullet-proof
- **Why:** The prefilter ran against `title + text`, but most sources carry thin or absent body text (Stack Exchange returns no `body`, many HN stories have empty `story_text`, Reddit is title-heavy), so the gate effectively required an emotional pain word (`hate`, `frustrat`, `boilerplate`, …) in the *title alone*. Real pain posts rarely phrase it that way, so the filter discarded almost everything before the LLM ever saw it — the 2026-05 evidence file held a single scored item across all four personas. The LLM scorer with the relevance floor is a far better topical judge, and it's the gate SK-ICP-002 already relies on. Removing the regex is also a net code deletion (P5).
- **Consequence in code:** `apps/api/src/icp-score.ts` loses the `PAIN_REGEX` constant and the `candidates` filter; the batch loop iterates `items` directly and returns `{ scored: items.length, skipped: 0, stored }` (or `skipped: items.length` when no LLM key). `test/icp-score.test.ts` replaces the prefilter-skip test with one asserting neutral-titled items still reach the LLM. Volume stays inside [`GLOBAL-013`](../../../decisions/GLOBAL-013-free-tier-bundle-budget.md): the source caps total ≲700 items/week ⇒ ≲35 Groq batch calls/week, well within the free tier.
- **Alternatives rejected:** Broaden the regex word list (still a brittle keyword gate that under-samples quiet, title-only pain — the LLM already judges relevance); keep the prefilter but only for body-bearing sources (added branching for no benefit at this volume); raise `RELEVANCE_FLOOR` to compensate for more noise (the floor already discards off-topic items; raising it would re-introduce the same false-negative problem the regex caused).

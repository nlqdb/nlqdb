# SK-PIVOT-014 — Home-flow reposition: the home's proof leads the wedge, not just its headline

- **Decision:** The home (`nlqdb.com/`) is rebuilt so the **flow and the proof** lead the agent-memory wedge end-to-end — closing the gap WS-13 / SK-PIVOT-013 left. The hero's **primary action** is connect-to-your-agent (the SK-WEB-016 `<McpInstall>` row of host buttons, per SK-WEB-017); the **`Demo.astro` beat** shows the agent loop (`nlqdb_remember` writes a typed row → `nlqdb_recall` asks a natural-language question → compiled GROUP BY SQL + answer rows) instead of the prior generalist orders-table `<nlq-data>` demo; the **`Replaces.astro` beat** swaps to the agent-memory replacement set (vector store top-k · per-agent SQLite · embedding cache · schema design · re-rank pipeline) collapsing into the `nlqdb_recall(…)` punchline. The generalist describe→table→embed story is **demoted to one quiet `.alsoworks` line** below the three beats, linking `/app/new` (GLOBAL-036's dual-front-door seam — one click away, never on the main canvas). The SK-WEB-002 goal input is **retained** in the hero as a secondary affordance so the no-signup-wall first-value promise (GLOBAL-007) and the morph-to-chat behaviour still hold.

- **Core value:** Goal-first, Honest latency, Creative

- **Why:** WS-13 / SK-PIVOT-013 was the **single** founder-gated worksheet of the pivot — it swapped the four lead strings (wordmark tagline, `README` H1, `llms.txt` lede, `SoftwareApplication` JSON-LD description) to lead with the agent-memory wedge. By design, it left the home's **primary action** and **demo** out of scope so the reposition stayed additive and one-commit-revertable (GLOBAL-036's rejected "full reposition now" alternative). Founder feedback after WS-13 landed: the home reads as incoherent — *"the title talks about agent memory and then there is a hero with input box which is actually for web chat or more observability even, and then there is a box with a table which is unclear what generated it, and then there is html tags"*. That mismatch is exactly the risk the worksheet sequence was hedging against; the hedge succeeded (this fix is two commits, no revert needed), and now the follow-on can land: WS-14 finishes the dual-front-door rebalance by making the home's **proof** match its headline. The text input itself reads as "web chat" because a bare goal-input IS the generalist on-ramp; for an agent builder the right first move is connecting nlqdb to their agent, not typing a sentence — so the SK-WEB-016 install row leads and the input demotes without being deleted (the morph-to-chat is the most-tested affordance on the site; demoting beats removing).

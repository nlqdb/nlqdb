# SK-WEB-014 — Homepage declares its brand entity: Organization + WebSite JSON-LD, no SearchAction

- **Decision:** `nlqdb.com/` (homepage only) emits two site-wide entity nodes via `apps/web/src/lib/site-jsonld.ts`: an `Organization` (stable `@id` `https://nlqdb.com/#organization`, `name`, `url`, `logo`, GitHub `sameAs`) and a `WebSite` (`@id` `#website`, `name`, `url`, `publisher` → the Organization `@id`). Every page's existing `SoftwareApplication` node names the same Organization as `publisher` by that `@id` (the fragment is the shared `ORGANIZATION_ID_FRAGMENT` constant, so the reference can't drift). The site-wide nodes are emitted only when `canonicalURL.pathname === "/"` — root-only, never per-page. **No `SearchAction` is emitted** until a GET query entrypoint exists.
- **Core value:** Effortless onboarding (distribution / AEO)
- **Why:** The per-page `SoftwareApplication` declared the product but no node declared the *brand entity*. `Organization` binds the "nlqdb" string to one canonical entity so Google and answer engines attribute brand queries to it; `WebSite.name` is the string Google may surface as the SERP site name. Declaring them once on the root with stable `@id`s — and referencing that Organization `@id` from every page's `SoftwareApplication.publisher` — lets crawlers consolidate one entity across the whole site from a single fetch, mirroring the ItemList/Breadcrumb pattern (`SK-CMP-006`, shared `lib/`). A `SearchAction` is deliberately omitted: the schema.org sitelinks-searchbox contract requires the target URL to *run* the query, but the goal-first hero submits via JS to `/v1/ask` (`SK-WEB-002`) and no GET route consumes a `q` term — a SearchAction pointing at a URL that ignores the param is a broken signal, so it stays out until a URL-driven query entrypoint exists.
- **Consequence in code:** `apps/web/src/lib/site-jsonld.ts` exports `organizationJsonLd`, `webSiteJsonLd`, and `ORGANIZATION_ID_FRAGMENT`; `Base.astro` builds the per-page `publisher` `@id` from that constant and emits the two entity nodes only on `/`. Adding a `SearchAction` is forbidden until a GET query route consumes the term. Covered by `site-jsonld.test.ts` (including a regression test that `potentialAction` stays absent).
- **Alternatives rejected:**
  - "Emit Organization/WebSite on every page" — site-wide nodes belong on the root; repeating them per-page adds bytes and a drift surface for no extra signal.
  - "Inline the Organization node into the per-page `SoftwareApplication` `publisher`" — duplicates the entity on every page instead of one node referenced by `@id`; defeats consolidation.
  - "Ship a SearchAction now pointing at the hero URL" — the URL ignores any `q` term, so it advertises a search the site can't perform — a broken structured-data signal.

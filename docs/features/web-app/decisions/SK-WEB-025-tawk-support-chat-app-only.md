# SK-WEB-025 — Tawk.to support chat on the product `/app` surfaces only

- **Decision:** The Tawk.to live-chat widget loads on the `/app` product surfaces only (`SupportChat.astro`, mounted beside `<AppAnalytics />` on `/app`, `/app/new`, `/app/connect`, `/app/keys`), never on marketing/blog/vs/solve. It is the official async embed snippet verbatim (the property ID in the `embed.tawk.to` URL is publishable by design — it grants chat access, nothing else). Tawk's three hosts (`embed`/`va`/`cdn.tawk.to`) are denylisted in `lib/boot-fallback.ts#EXTENSION_PREFIXES` (plus the `Base.astro` hand-copy, drift-pinned by `boot-fallback.test.ts`) so a Tawk-side throw never paints the boot-fallback panel or spams `/v1/errors/web`; the widget loading async means its failure leaves the app pages fully functional.
- **Core value:** Free, Effortless UX, Bullet-proof
- **Why:** A signed-in user hitting a wall inside the product is the highest-signal pre-PMF inbound moment (`docs/founder-playbook.md` inbound-triage SLA); an in-page chat removes the "open GitHub, write an issue" detour. Tawk.to is the free, mature option per `GLOBAL-033` (unlimited agents/chats at $0). Scoping to `/app` preserves the marketing Lighthouse-100 + no-cookie-banner posture (`GLOBAL-034`) — Tawk sets its own cookies, so it must never ship on public pages.
- **Consequence in code:** `apps/web/src/components/SupportChat.astro` included by the four `/app` pages; Tawk hosts appended to `EXTENSION_PREFIXES` in `apps/web/src/lib/boot-fallback.ts` and the `Base.astro` inline copy; user-facing disclosure in `apps/web/src/pages/privacy.astro`, channel policy in `SUPPORT.md`, sub-processor row in `SUBPROCESSORS.md`. Adding a new `/app` page includes `<SupportChat />` next to `<AppAnalytics />`; a marketing page must not.
- **Alternatives rejected:**
  - **Site-wide embed in `Base.astro`.** Ships a cookie-setting third-party script to every marketing page — breaks the `GLOBAL-034` posture and the privacy page's no-tracking-cookies stance.
  - **Intercom / Crisp.** Free tiers cap seats/features that Tawk gives away; fails strict-$0 (`GLOBAL-033`).
  - **No chat (issues/discussions only).** Loses the stuck-in-product inbound moment; a GitHub round-trip filters out exactly the confused-user signal the founder needs.

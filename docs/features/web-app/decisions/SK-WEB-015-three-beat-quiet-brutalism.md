# SK-WEB-015 — Three-beat homepage + quiet-brutalism token system

> **Status:** the **token system** (neutrals, acid-lime accent, serif/mono faces, 0 radius, hard offset shadows) is **superseded by [`SK-WEB-020`](./SK-WEB-020-calm-token-system.md)** (the calm system) site-wide; the three-beat `/` IA was already superseded by [`SK-WEB-018`](./SK-WEB-018-two-door-home.md). What carries forward into SK-WEB-020: one accent moment per band, mono demoted to code/data, one token source in `global.css`, and the one-motion-moment / `prefers-reduced-motion` budget. The values named below are historical.

- **Decision:** The marketing site adopts a single design system ("quiet brutalism") and a three-beat homepage IA. **Tokens** live in `apps/web/src/styles/global.css` as the only source of design values: six warm-green-biased neutrals (`--ground #0b0f0a`, `--raised #161b14`, `--rule #2a2f26`, `--muted #8b8b85`, `--body #c8c8c0`, `--ink #f4f4f0`); one accent (`--accent #c6f432`, `--accent-ink #0b0f0a`) **reserved for one moment per fold** (hero connect-row CTA → the two MCP tool identifiers in the demo → final CTA — three lime moments total on `/`); three faces (`--display` Source Serif 4 for narrative headlines, `--mono` JetBrains Mono restricted to code/data/identifiers, `--sans` Inter for body); five type steps (`--t-display`, `--t-section`, `--t-large`, `--t-body`, `--t-small`); two widths (`--col 720px`, `--bleed 1120px`); two gaps (`--gap-beat`, `--gap-block`). Page-specific styling lives in scoped `<style>` blocks per `.astro` file; pages must not redefine tokens or add new ones. **Homepage IA** is exactly three beats: (1) **WHAT** — wordmark in nav, display-serif headline + one-line lede + the hero's primary action (the SK-WEB-016 `<McpInstall compact>` row per SK-WEB-017; the SK-WEB-002 goal input is retained as a quieter secondary below it); (2) **HOW** — `Demo.astro` showing the agent loop server-side (`nlqdb_remember` writes a typed row → `nlqdb_recall` asks a NL question → compiled GROUP BY SQL + answer rows; replaces the prior `<nlq-data>` orders-table demo so the proof matches the agent-memory headline, closing the WS-13 gap recorded in SK-PIVOT-014); (3) **WHY** — `Replaces.astro` in editorial serif, the agent-memory strikethrough list (vector top-k · per-agent SQLite · embedding cache · schema design · re-rank pipeline) collapsing into the `nlqdb_recall(…)` punchline + one CTA below. A single quiet `.alsoworks` line below the three beats links the generalist describe→table→embed flow to `/app/new` (GLOBAL-036 dual-front-door demotion: one click away, never on the main canvas). `AgentMemoryBand`, `AlsoWorksFor`, `ResearchReceipts`, `ManifestoExcerpt` are removed from `/`. **Page-by-page treatment:** `/agents` mirrors the three-beat rhythm with agent-memory copy; `/manifesto` is a single 720px editorial column (drop cap, pull-quotes, no hard-shadow boxes); `/pricing` uses one button primitive across all tiers, prices in mono; `/vs/*` and `/solve/*` collapse to *what-we-replace → same-goal-two-ways → tight 4-row diff*; `/privacy` and `/terms` get the token sweep only. **Motion budget:** maximum one intentional motion moment per page, gated on `@media (prefers-reduced-motion: no-preference)`.
- **Core value:** Effortless UX, Goal-first, Honest latency
- **Why:** The pre-redesign homepage stacked four text layers before the input, ran ten sections below the fold, mixed three competing fluid type scales, and used lime on the wordmark, CTAs, links, dividers, and decorative shapes simultaneously — so no accent landed and no story arc cohered. The 2026-06-24 thesis pivot to "analytical memory for AI agents" (WS-13 / SK-PIVOT-013) shipped on top of the old layout, leaving the credo line apologising for the headline. The fix is not more design but **less page with restraint**: one sentence ("nlqdb is durable memory your agents query in English") delivered by three beats and nothing else. Demoting mono to code/data only is the single biggest readability gain available because today everything looks like code so nothing reads as code. Reserving lime to three moments per fold restores the accent's signal value. A single token system removes the "different font sizes and indented" complaint at its root by collapsing many independent scales into six neutrals / three faces / five steps / two widths / two gaps. Every honoured invariant (`SK-WEB-002` one input, `SK-WEB-003` runnable code above the fold, `SK-WEB-008` real `/v1/ask`, `SK-WEB-010` Copy-snippet shape, `SK-WEB-014` JSON-LD, `GLOBAL-007` no login wall, `GLOBAL-011` honest latency, `GLOBAL-012` one-sentence errors, `GLOBAL-020` no first-60s config, `GLOBAL-023` trust UX, `GLOBAL-025` UX pillar advanced without degrading engine/onboarding/perf — smaller page is a perf gain) is preserved or strengthened.
- **Consequence in code:** `apps/web/src/styles/global.css` is the single source of token values and may not be modified by page work; all page-specific styling lives in scoped `<style>` blocks in `*.astro` files using only these tokens. `apps/web/src/pages/index.astro` ships three components plus one quiet `.alsoworks` line in order: hero (wordmark in nav, no credo line, no lime on wordmark; `<McpInstall compact>` connect row primary + retained `<CreateForm>` secondary, per SK-WEB-017), `Demo.astro` (the agent loop server-side; lime moment = the two MCP tool identifiers), `Replaces.astro` (editorial-serif strikethrough list + one CTA), then the `.alsoworks` line linking the generalist flow to `/app/new`. `AgentMemoryBand.astro`, `AlsoWorksFor.astro`, `ResearchReceipts.astro`, `ManifestoExcerpt.astro` are removed from `/` (their content relocates: `AgentMemoryBand` folds into the demo's contextual copy with `/agents` as the destination, `AlsoWorksFor` moves to footer/`/integrations`, `ResearchReceipts` moves to `/manifesto`, `ManifestoExcerpt` is dropped — the footer manifesto link is enough). `/agents`, `/manifesto`, `/pricing`, `/vs/*`, `/solve/*`, `/privacy`, `/terms` follow the page-by-page treatment above. Source Serif 4 is loaded globally (Latin subset, variable weights 400–700, `font-display: swap`, ~50 KB woff2). Every animation is gated on `prefers-reduced-motion: no-preference`; no page introduces a second intentional motion moment. New pages and PRs that redefine tokens, add ad-hoc scales, place lime outside the three permitted moments per fold, or reintroduce the cut homepage sections are rejected.
- **Alternatives rejected:**
  - **Sharpen brutalism — keep mono as the narrative voice, fix only the rhythm.** Rejected because the diagnosis "everything looks like code so nothing reads as code" cannot be resolved without a non-mono companion face; tightening rhythm alone leaves the readability ceiling in place.
  - **Light reset — invert to a near-white ground.** Rejected because a near-black ground is the brand's most recognisable property in screenshot / share contexts and no audience signal asks for it; the cost (brand reset) is real, the benefit is speculative.
  - **Keep the ten-section homepage and tighten copy only.** Rejected because the diagnosis is the page itself, not the words on it; ten sections cannot deliver a one-sentence thesis no matter how the copy is tuned.
  - **Use lime broadly (current state) for "energy".** Rejected because loud accents only land when surrounded by quiet — broad lime use is why the existing CTAs read as decoration rather than action.

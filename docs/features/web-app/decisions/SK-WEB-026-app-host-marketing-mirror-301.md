# SK-WEB-026 — Merged app host 301-redirects the whole marketing surface to the canonical host

- **Decision:** The merged app host `app.nlqdb.com` (`nlqdb-api`, `SK-AUTH-016`) serves the **same** `apps/web/dist` build as the canonical marketing host `nlqdb.com`, so its entire marketing surface is a crawlable duplicate. A thin front-controller in `apps/api/src/index.ts` (`marketing-mirror.ts`) **301-redirects that surface on `app.nlqdb.com` to `https://nlqdb.com/<same-path>`**, consolidating SEO authority on the one canonical host. The matched set is `MARKETING_MIRROR_PREFIXES`: the content trees (`/blog`, `/solve`, `/vs`), the static marketing singles (`/agents`, `/architecture`, `/integrations`, `/manifesto`, `/pricing`, `/privacy`, `/terms`, `/security`), and the SEO/discovery aggregators (`/llms.txt`, `/rss.xml`, `/sitemap.xml`). The product (`/app/*`), auth (`/auth/*`, `/oauth/*`), API (`/v1/*`, `/api/auth/*`), and the root `/` are untouched — the app host's own routes. `apps/api/wrangler.toml` scopes `run_worker_first` to exactly these paths so every other path keeps its zero-cost asset-first / Hono route. On preview / `*.workers.dev` hosts the front-controller serves the built asset (no duplication to fix there, and Hono has no marketing routes).
  - **Superseded 2026-07-21 (run 109):** the original scope was **trees only** — singles and aggregators were excluded as "a far smaller duplicate surface" kept out for list stability. Fresh GSC then showed the excluded singles indexed with impressions (`/agents` pos 7.3, `/architecture` pos 12.3) and the app host still 200-serving `/pricing`, `/manifesto`, `/sitemap.xml`, `/rss.xml`, `/llms.txt`. Since the app host "has nothing of its own to rank" on any of these (the core rationale below), the trees-only line was scoping convenience, not a boundary — coverage is now the whole surface. The stability concern is addressed by enforcement, not exclusion (see Consequence).
- **Core value:** Free, Simple, Bullet-proof
- **Why:** GSC (2026-07) surfaced `app.nlqdb.com/blog/postgres-validator-rejects-valid-clickhouse-sql/` indexed at page-1 (position 8) — a duplicate of the canonical `nlqdb.com/blog/…`. The pages already carry a correct `rel=canonical` to `nlqdb.com` and the sitemap lists only `nlqdb.com` URLs, yet Google indexed the app-host copy anyway (`rel=canonical` is a hint, not a directive). As indexation grows across the 115 submitted URLs, a full second crawlable host splits crawl budget and ranking authority — directly suppressing the canonical host's impression breadth, which is the acquisition bottleneck (scorecard row #7). A 301 is the one signal Google treats as authoritative for host consolidation. The app host has nothing of its own to rank (product is behind auth; API returns JSON), so redirecting its marketing trees has no downside.
- **Consequence in code:** `apps/api/src/marketing-mirror.ts` exports the pure `marketingMirrorRedirect(url)` + `isMarketingMirrorPath(pathname)` (unit-tested in `marketing-mirror.test.ts`); the worker `export default.fetch` calls them before `app.fetch`. `apps/api/wrangler.toml` `[assets]` carries a `run_worker_first` array derived from `MARKETING_MIRROR_PREFIXES` (each non-file prefix → its bare entry + `/*` glob; each aggregator file → its exact entry). Adding a NEW top-level marketing route is a **one-list edit** — add its prefix to `MARKETING_MIRROR_PREFIXES` and the matching `run_worker_first` entries; a `marketing-mirror.test.ts` guard parses the toml and **fails if the two lists drift**, so a forgotten `run_worker_first` entry (which would silently re-expose the route as a crawlable duplicate) can't ship. This enforced sync is why the surface can now be complete rather than trees-only. Consistent with `SK-WEB-023`'s index-open posture — it *enforces* the self-referential canonical by removing the non-canonical host, rather than adding a `noindex`.
- **Alternatives rejected:**
  - `X-Robots-Tag: noindex` on the app host's marketing pages — needs the same `run_worker_first` intercept but leaves the duplicate URLs live (weaker than a 301, which forwards any inbound link equity to the canonical).
  - `robots.txt` `Disallow` on the app host — the file is shared static (`../web/dist`), so it can't be host-differentiated without the same worker intercept; and `Disallow` blocks crawl so Google can't even see the canonical, leaving stale index entries. Google's own guidance prefers 301 / `noindex` over `Disallow` for de-duplication.
  - A Cloudflare dashboard Redirect Rule — a console click; not agent-movable (`GLOBAL-033`).
  - `run_worker_first = true` (worker in front of every request) — routes all product-asset traffic through the Hono app; larger blast radius for a bounded SEO fix.

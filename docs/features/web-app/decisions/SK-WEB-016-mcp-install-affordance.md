# SK-WEB-016 ‚Äî One-click MCP install affordance: shared `<McpInstall>` at three venues, deep-link where supported

- **Decision:** Ship a single `<McpInstall apiKey mcpUrl compact />` Astro component (`apps/web/src/components/McpInstall.astro`) that renders four host buttons ‚Äî **Claude ¬∑ Cursor ¬∑ Windsurf ¬∑ Zed** ‚Äî and place it at three venues: (a) `/agents` final connect card (replacing the current copy-config / CLI two-card grid), (b) the marketing post-create `CreateResultView` alongside the existing `<CreateSnippetView>` (`SK-WEB-010`), and (c) the `/integrations` MCP-hosts section (replacing the static four-cell informational grid). Where a host ships a documented deep-link install scheme, the button is an `<a>` whose `href` is the resolved URI (`target="_self"`, the OS hands the click to the host's URI handler); where no scheme exists, the button copies a paste-ready JSON config block and labels the one place to paste it. **Per-host status (canonical, Phase 1 research):** Cursor ‚Üí deep-link `cursor://anysphere.cursor-deeplink/mcp/install?name=nlqdb&config=<base64-url-encoded-JSON>` where the JSON is the inner `mcpServers` value `{"url":"https://mcp.nlqdb.com"}`. Claude Desktop ‚Üí fallback-only (Custom Connectors via Settings; no documented `claude://` install scheme as of Jun 2026, verified against Anthropic support docs + `anthropics/claude-code#26952`); copies `{"mcpServers":{"nlqdb":{"url":"https://mcp.nlqdb.com"}}}`. Windsurf ‚Üí fallback-only (`windsurf://windsurf-mcp-registry?serverName=‚Ä¶` exists but only opens the registry page for already-published servers, not arbitrary URLs); copies `{"mcpServers":{"nlqdb":{"serverUrl":"https://mcp.nlqdb.com/mcp"}}}`. Zed ‚Üí fallback-only (no scheme shipped; `zed://mcp/install` is open feature request `zed-industries/zed#58096` only); copies `{"context_servers":{"nlqdb":{"url":"https://mcp.nlqdb.com"}}}` (note: Zed's wrapper key is `context_servers`, NOT `mcpServers`). One promoted lime button per row (`SK-WEB-015` accent budget ‚Äî "Add to Claude" wins by default since Claude is the first-party hosted MCP target); every other button is `.cta--ghost`. **Anonymous-key contract:** the `apiKey` prop defaults to the literal `"pk_live_REPLACE_ME"` (`SK-WEB-010`); when it equals the placeholder, the component renders a one-line *"Sign in (free) to inline your live key ‚Üí"* footnote linking to `/auth/sign-in?return_to=/app`. When a real `pk_live_‚Ä¶` is passed (post-sign-in surfaces), the footnote is hidden. **Compact prop:** `compact={true}` packs the four buttons into a single flex row with terse labels for the post-create surface; `compact={false}` lays them out in a responsive grid with descriptive subline copy for `/agents` and `/integrations`. **Motion budget:** the transient "Copied ‚úì" state is gated on `@media (prefers-reduced-motion: no-preference)` per `SK-WEB-015`.

- **Core value:** Effortless UX, Goal-first, Seamless auth

- **Why:** The hero ends with *"you have a database."* The next question every agent builder asks ‚Äî within seconds ‚Äî is *"how does my agent talk to it?"* Today the answer is three friction-laden steps: read the docs, find the connector URL, paste it into the host's config file (in the right shape, in the right location, against the right wrapper key). Most agent IDEs now ship a deep-link install scheme exactly to collapse that to one click; for the hosts that don't, copying a paste-ready JSON block (with the correct wrapper key and field name for *that specific host* ‚Äî `mcpServers` vs `context_servers`, `url` vs `serverUrl`) is the next-best step. A single `<McpInstall>` component encapsulates the per-host shape knowledge so a docs-stale rendering of the install path can never ship from a one-off page edit. Three venues are load-bearing: `/agents` is the page whose whole pitch is *agent memory over MCP* ‚Äî install adjacency is on-message, not interruption (the existing connect card already implies this; `SK-WEB-016` formalises it with the deep-link buttons replacing the manual copy-config UI). The post-create surface catches the user at peak engagement ‚Äî they just saw their schema render ‚Äî and converts "I have a DB" into "my agent can talk to it" without leaving the page. `/integrations` is the canonical reference page; replacing the static four-cell host grid with the installer turns documentation into action. `SK-WEB-002` (one input above the fold) is preserved ‚Äî the install row sits *after* the CTA on `/agents`, never beside it, and never appears on the marketing homepage. `SK-WEB-015` accent budget is preserved ‚Äî one promoted lime CTA per row, the rest `.cta--ghost`. `SK-ANON-012`'s no-live-anon-key contract is preserved ‚Äî the placeholder key + sign-in CTA is the honest path (inlining a real anon key would 401 because the 1-call cap was already burned by the create call). `SK-WEB-007`'s rule that only the chat inlines a working key is preserved ‚Äî the McpInstall configs ship the placeholder until a real `pk_live_` arrives from the per-DB key path.

- **Consequence in code:** New `apps/web/src/components/McpInstall.astro` is the only place the per-host install knowledge lives ‚Äî the four host entries (`id`, `status`, `href` or `config`, `pasteHint`, `versionHint`) sit in one `HOSTS` array at the top of the file. The Cursor deep-link is built at SSR via `Buffer.from(JSON.stringify({url}), "utf8").toString("base64")` then `encodeURIComponent`'d into the `config` query parameter. Each fallback config is the FULL paste-ready document for that host (correct wrapper key per host ‚Äî `mcpServers` for Claude/Windsurf, `context_servers` for Zed; correct URL field ‚Äî `url` for Claude/Zed, `serverUrl` for Windsurf with the `/mcp` path suffix). Clipboard handling is vanilla JS inside an inline `<script>` (no React island ‚Äî the component ships zero JS framework weight); failures are swallowed silently. The success animation is one motion moment gated on `prefers-reduced-motion: no-preference` (`SK-WEB-015`). The demand signal reuses the existing `home.snippet_copied` LogSnag envelope with `surface = mcp_install_<host>` so the install funnel reads on the same dashboard as the embed-snippet funnel (`SK-WEB-010` / `GLOBAL-024`). Three venues land in stacked PRs after the foundation: (1) `apps/web/src/pages/agents/index.astro` swaps the existing `.ag-connect__grid` (currently a copy-config card + a CLI card) for `<McpInstall compact={false} />` plus the same CLI escape hatch below; (2) `apps/web/src/components/CreateForm.tsx` renders `<McpInstall compact={true} apiKey="pk_live_REPLACE_ME" />` inside `CreateResultView` after `<CreateSnippetView>` (the component is server-rendered through an Astro wrapper since CreateForm is a React island ‚Äî wired via a slot or a thin React proxy in the next stack); (3) `apps/web/src/pages/integrations.astro`'s `ints-mcp` section becomes the component (the four-cell informational grid drops). PRs that add a fifth host without re-running the Phase 1 research (deep-link scheme + min version + fallback config shape) are rejected ‚Äî every host entry must cite its source. PRs that move lime onto a second button in the row are rejected ‚Äî one promoted CTA per row is the `SK-WEB-015` accent ladder.

- **Alternatives rejected:**
  - **Render buttons next to the homepage hero input.** Rejected because `SK-WEB-002` keeps the hero to one input above the fold; the install row would compete with the "what are you building?" affordance and dilute the goal-first inversion. The post-create surface (which IS after the CTA) is the right venue for the marketing flow.
  - **Multi-step text instructions, one set per host.** Rejected because the per-host config shape is non-trivially divergent (Claude's `mcpServers` + `url`, Windsurf's `mcpServers` + `serverUrl` + `/mcp` suffix, Zed's `context_servers` + `url`, plus three different paste locations), and host versions ship breaking changes to those shapes regularly ‚Äî a copy-paste-correct-shape button captures the divergence in one place where it can be updated once.
  - **Inline a real anon `pk_live_‚Ä¶` in the McpInstall configs.** Rejected because `SK-ANON-012`'s per-device cap is consumed by the create call; the anon key would 401 on the first tool call from the host. Silent-broken-install is a worse UX than the placeholder + honest sign-in nudge.
  - **One button per host with the same label "Install" and a host icon.** Rejected because the label "Add to Claude" / "Add to Cursor" carries the host name as scannable text ‚Äî icons read worse on screen readers and lose the brand recognition cue that drives the click. The version hint (e.g. "Cursor current stable, 2026+") rides the `title` attribute so the affordance stays honest without inflating the row's visual weight.
  - **Publish to the Windsurf MCP Registry and use `windsurf://windsurf-mcp-registry?serverName=nlqdb` as the deep-link path.** Rejected for the foundation ‚Äî the registry submission is a separate workstream (review process, ownership, takedown risk). Once published, swapping Windsurf from `status: "fallback-only"` to `status: "deep-link"` is a one-line change in `HOSTS`.

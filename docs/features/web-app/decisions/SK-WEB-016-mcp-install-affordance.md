# SK-WEB-016 — One-click MCP install affordance: shared `<McpInstall>` at three venues, deep-link where supported

- **Decision:** Ship a single `<McpInstall apiKey mcpUrl compact />` Astro component (`apps/web/src/components/McpInstall.astro`) that renders a row of host buttons — **Cursor · VS Code · Claude Code · Codex · Claude Desktop · Windsurf · Zed** — and place it at four venues: (a) `/agents` final connect card (replacing the current copy-config / CLI two-card grid), (b) the marketing post-create `CreateResultView` alongside the existing `<CreateSnippetView>` (`SK-WEB-010`), (c) the `/integrations` MCP-hosts section (replacing the static informational grid), and (d) **Door A of the two-door home** (`SK-WEB-018`). Three install affordances exist, one per host descriptor `status`: a host that ships a documented deep-link install scheme is `status: "deep-link"` — the button is an `<a>` whose `href` is the resolved URI (`target="_self"`, the OS hands the click to the host's URI handler); a host driven by a one-line CLI install command is `status: "command"` — clicking copies that command and reveals it in place; a host with neither is `status: "fallback-only"` — clicking copies a paste-ready JSON config block and labels the one place to paste it. **Click→reveal-in-place is the uniform behaviour:** every host's click reveals its fallback install instructions (the JSON config, the command, or the paste location) inline below the row *and*, for `deep-link`/`command` hosts, fires the open/install so the user always has the manual fallback even when the automatic path runs. **Per-host status (canonical, Phase 1 research):** every config points at the full endpoint `https://mcp.nlqdb.com/mcp` — the server serves the MCP protocol at `/mcp`, not root, so a bare-domain config 404s on the first POST (`MCP_ENDPOINT_URL` in `lib/mcp-install.ts`, pinned by `mcp-install.test.ts`). Cursor → `deep-link` `cursor://anysphere.cursor-deeplink/mcp/install?name=nlqdb&config=<base64-url-encoded-JSON>` where the JSON is the inner `mcpServers` value `{"url":"https://mcp.nlqdb.com/mcp"}`. **VS Code → `deep-link`** `vscode:mcp/install?<url-encoded-JSON>` (the `vscode-insiders:` variant for Insiders). **Claude Code → `command`** (`claude mcp add --transport http nlqdb https://mcp.nlqdb.com/mcp`). **Codex → `command`** (the OpenAI Codex CLI `codex mcp add` form). Claude Desktop → `fallback-only` (Custom Connectors via Settings; no documented `claude://` install scheme); copies `{"mcpServers":{"nlqdb":{"url":"https://mcp.nlqdb.com/mcp"}}}`. Windsurf → `fallback-only` (`windsurf://windsurf-mcp-registry?serverName=…` only opens the registry page for already-published servers, not arbitrary URLs); copies `{"mcpServers":{"nlqdb":{"serverUrl":"https://mcp.nlqdb.com/mcp"}}}`. Zed → `fallback-only` (`zed://mcp/install` is open feature request `zed-industries/zed#58096` only); copies `{"context_servers":{"nlqdb":{"url":"https://mcp.nlqdb.com/mcp"}}}` (note: Zed's wrapper key is `context_servers`, NOT `mcpServers`). **The exact deep-link URI scheme, command syntax, and minimum host version for every entry above MUST be cited from each host's current docs at implementation time** — the strings here are the shape; the implementing stream fills the receipts (and the `mcp-install.test.ts` source assertions) before shipping a host, per the no-unsourced-host rule below. One promoted lime button per row (`SK-WEB-015` accent budget — the first-party hosted MCP target wins the promotion by default); every other button is `.cta--ghost`. **Anonymous-key contract:** the `apiKey` prop defaults to the literal `"pk_live_REPLACE_ME"` (`SK-WEB-010`); when it equals the placeholder, the component renders a one-line *"Sign in (free) to inline your live key →"* footnote linking to `/auth/sign-in?return_to=/app`. When a real `pk_live_…` is passed (post-sign-in surfaces), the footnote is hidden. **Compact prop:** `compact={true}` packs the buttons into a single flex row with terse labels for the post-create + home-door surfaces; `compact={false}` lays them out in a responsive grid with descriptive subline copy for `/agents` and `/integrations`. **Motion budget:** the transient "Copied ✓" state is gated on `@media (prefers-reduced-motion: no-preference)` per `SK-WEB-015`.

- **Core value:** Effortless UX, Goal-first, Seamless auth

- **Why:** The hero ends with *"you have a database."* The next question every agent builder asks — within seconds — is *"how does my agent talk to it?"* Today the answer is three friction-laden steps: read the docs, find the connector URL, paste it into the host's config file (in the right shape, in the right location, against the right wrapper key). Most agent IDEs now ship a deep-link install scheme exactly to collapse that to one click; for the hosts that don't, copying a paste-ready JSON block (with the correct wrapper key and field name for *that specific host* — `mcpServers` vs `context_servers`, `url` vs `serverUrl`) is the next-best step. A single `<McpInstall>` component encapsulates the per-host shape knowledge so a docs-stale rendering of the install path can never ship from a one-off page edit. Three venues are load-bearing: `/agents` is the page whose whole pitch is *agent memory over MCP* — install adjacency is on-message, not interruption (the existing connect card already implies this; `SK-WEB-016` formalises it with the deep-link buttons replacing the manual copy-config UI). The post-create surface catches the user at peak engagement — they just saw their schema render — and converts "I have a DB" into "my agent can talk to it" without leaving the page. `/integrations` is the canonical reference page; replacing the static four-cell host grid with the installer turns documentation into action. `SK-WEB-002` (one input above the fold) is preserved — the install row sits *after* the CTA on `/agents`, never beside it, and never appears on the marketing homepage. `SK-WEB-015` accent budget is preserved — one promoted lime CTA per row, the rest `.cta--ghost`. `SK-ANON-012`'s no-live-anon-key contract is preserved — the placeholder key + sign-in CTA is the honest path (inlining a real anon key would 401 because the 1-call cap was already burned by the create call). `SK-WEB-007`'s rule that only the chat inlines a working key is preserved — the McpInstall configs ship the placeholder until a real `pk_live_` arrives from the per-DB key path.

- **Consequence in code:** `apps/web/src/lib/mcp-install.ts` is the single source of truth for the per-host install knowledge — `buildMcpHosts(mcpUrl)` returns the seven entries (`id`, `status: "deep-link" | "command" | "fallback-only"`, `href` for deep-link, `command` for command, `config` + `pasteHint` for fallback-only, `versionHint`, `sourceUrl` — the doc the syntax was cited from), and exports `MCP_ENDPOINT_URL` / `MCP_SERVER_ROUTE` / `PLACEHOLDER_KEY` / `PROMOTED_HOST`. Clicking any host reveals its fallback (`config` / `command` / `pasteHint`) in place; `deep-link` and `command` hosts additionally trigger the open/copy so the manual fallback is always present. Both venues import it so they can't drift: `McpInstall.astro` (SSR, vanilla-JS clipboard handler, zero framework weight) and the `McpInstallView` React mirror inside `CreateForm.tsx` (the Astro component scopes its styles, so the React island can't borrow them and renders an equivalent surface inline). The Cursor deep-link is `base64(JSON.stringify({url: MCP_ENDPOINT_URL}))` URL-encoded into the `config` query param; `toBase64` feature-detects `Buffer` (SSR) vs `btoa` (browser) so both venues produce identical URIs. `mcp-install.test.ts` decodes every host's connection URL and asserts its path equals `MCP_SERVER_ROUTE` — catching root, a doubled `/mcp/mcp`, or any other drift. Clipboard failure is swallowed silently (the `/integrations` configs stay visible). The demand signal reuses the `home.snippet_copied` LogSnag envelope with `surface = mcp_install_<host>` so the install funnel reads on the embed-snippet dashboard (`SK-WEB-010` / `GLOBAL-024`). Venues: Door A of the two-door home (`SK-WEB-018`, compact), `/agents` hero (compact, under the form, `--rule` hairline above), post-create `CreateResultView` (after the schema preview, before `<CreateSnippetView>`), `/integrations` MCP-hosts section (`compact={false}`, the old grid dropped). **No-unsourced-host rule:** every host entry MUST carry a `sourceUrl` citing the current host doc its `href`/`command`/`config` shape came from, and `mcp-install.test.ts` asserts the endpoint path; PRs that add a host (or change a deep-link scheme / command) without re-running that research and updating the `sourceUrl` are rejected. PRs that move lime onto a second button per row are rejected — one promoted CTA per row is the `SK-WEB-015` accent ladder.

- **Alternatives rejected:**
  - **Render buttons next to the homepage hero input.** Rejected because `SK-WEB-002` keeps the hero to one input above the fold; the install row would compete with the "what are you building?" affordance and dilute the goal-first inversion. The post-create surface (which IS after the CTA) is the right venue for the marketing flow.
  - **Multi-step text instructions, one set per host.** Rejected because the per-host config shape is non-trivially divergent (Claude's `mcpServers` + `url`, Windsurf's `mcpServers` + `serverUrl` + `/mcp` suffix, Zed's `context_servers` + `url`, plus three different paste locations), and host versions ship breaking changes to those shapes regularly — a copy-paste-correct-shape button captures the divergence in one place where it can be updated once.
  - **Inline a real anon `pk_live_…` in the McpInstall configs.** Rejected because `SK-ANON-012`'s per-device cap is consumed by the create call; the anon key would 401 on the first tool call from the host. Silent-broken-install is a worse UX than the placeholder + honest sign-in nudge.
  - **One button per host with the same label "Install" and a host icon.** Rejected because the label "Add to Claude" / "Add to Cursor" carries the host name as scannable text — icons read worse on screen readers and lose the brand recognition cue that drives the click. The version hint (e.g. "Cursor current stable, 2026+") rides the `title` attribute so the affordance stays honest without inflating the row's visual weight.
  - **Publish to the Windsurf MCP Registry and use `windsurf://windsurf-mcp-registry?serverName=nlqdb` as the deep-link path.** Rejected for the foundation — the registry submission is a separate workstream (review process, ownership, takedown risk). Once published, swapping Windsurf from `status: "fallback-only"` to `status: "deep-link"` is a one-line change in `HOSTS`.

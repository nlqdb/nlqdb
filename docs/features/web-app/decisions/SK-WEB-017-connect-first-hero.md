# SK-WEB-017 — Connect-first hero on the agent-memory home; goal input retained as secondary

- **Decision:** The home (`nlqdb.com/`) hero leads with **connect-to-your-agent** — the SK-WEB-016 `<McpInstall compact>` row of host buttons is the primary affordance, introduced by *"Connect it to your agent."* The single goal text input (`<CreateForm>`) is **retained** in the hero as a quieter secondary, separated by a `--rule` line and introduced by *"Or spin one up yourself —"*. The text input still posts to `/v1/ask` with an anonymous bearer (SK-ANON-008), still morphs in place to the chat (SK-WEB-002's morph-to-chat behaviour), and still requires no sign-in (GLOBAL-007). Only its **visual primacy** is demoted; its **functional contract is unchanged**. The same SK-WEB-002 split applies on `/agents` (where it already does).

- **Core value:** Goal-first, Effortless UX, Honest latency, Bullet-proof

- **Why:** WS-13 / SK-PIVOT-013 swapped the headline strings on `/` to *"Analytical memory for AI agents."* but left the hero's **primary action** — the lone goal input — showing the generalist describe→table→embed flow. Founder feedback this session: that mismatch is what made the home read as incoherent — the title talked about agent memory, the screen below it demonstrated a different product. An agent builder lands on a wedge headline and the first thing the eye finds should be *how* they connect nlqdb to their agent, not a textbox. The `<McpInstall>` row built in SK-WEB-016 already does that work — it just wasn't in the hero. This decision is the one-line reposition: the install row leads, the input demotes; the home now tells one story end to end. The text input is kept (not deleted) because (a) `GLOBAL-007` requires no-login-wall first value, and the anon-create flow IS that, and (b) the morph-to-chat is the most-tested affordance on the site — removing it for a softer message is a worse trade than demoting it.

- **Consequence in code:** `apps/web/src/components/Hero.astro` renders, in order: display-serif lede → one sans-serif explainer → `<div class="hero__connect">` containing the *"Connect it to your agent."* eyebrow + `<McpInstall compact={true} />` → `<div class="hero__try">` (separated by a `border-top: 1px solid var(--rule)`) containing the *"Or spin one up yourself —"* label + `<CreateForm>`. The form submit CTA keeps the lime accent but its `font-size` is set to `--t-body` (a step smaller than the connect row) so the eye lands on connect first. No new tokens; SK-WEB-015 motion budget unchanged. `apps/web/src/pages/index.astro` adds a small `.alsoworks` line below the third beat (a quiet `--rule`-separated sentence linking the generalist describe→table→embed story to `/app/new`), honouring GLOBAL-036's dual-front-door demotion ("the generalist flow is one click away, never on the main canvas"). The home's JSON-LD `description` is rewritten to lead with the wedge string.

- **Alternatives rejected:**
  - **Keep SK-WEB-002 verbatim — only swap the demo.** Closes part of the gap (the demo would match the headline) but the hero's primary action would still be a goal textbox while the headline talks about agent memory; the founder feedback is specifically that the text input itself reads as "web chat" and confuses the message.
  - **Delete the goal input from the home entirely; make it `/app/new`-only.** Removes the no-login-wall first-value affordance on the highest-traffic surface; the morph-to-chat is the most-tested flow on the site. Demotion preserves both promises (GLOBAL-007 + SK-WEB-002's morph) for visitors who do want to type.
  - **Reframe the single input as agent-shaped** (e.g. *"What should your agent remember?"*) and skip the install row entirely. Leaves the developer with a text box to type into when their real next step is connecting nlqdb to their agent — wrong action for the audience.
  - **Connect row outside the hero** (e.g. beat 2 only). Breaks the "headline + first action match" the founder feedback explicitly called for; the user has to scroll past a fold to find the install path.

# SK-AUTH-019 ŌĆö `/api/auth/sign-out` bypasses `originCheckMiddleware` via direct `auth.api.signOut`

- **Decision:** A dedicated `app.post("/api/auth/sign-out")` in `apps/api/src/index.ts`, ahead of the Better Auth catch-all, calls `auth.api.signOut({ headers, request, asResponse: true })`. The direct-API path skips Better Auth's `routerMiddleware` chain (see `to-auth-endpoints.mjs`), so `originCheckMiddleware`'s `INVALID_ORIGIN` / `MISSING_OR_NULL_ORIGIN` 403s never fire on sign-out. D1 session delete, KV revocation (`SK-AUTH-007` / `databaseHooks.session.delete.after`), and cookie expiry all still run inside `auth.api.signOut`. Every other `/api/auth/*` endpoint stays on `auth.handler` and keeps the origin check.
- **Core value:** Bullet-proof
- **Why:** Authenticated POSTs to `/api/auth/sign-out` from `app.nlqdb.com` were 403'ing in production. Privacy extensions / corporate MITMs / mobile browsers strip the `Origin` header from same-origin POSTs often enough that gating sign-out on the origin check left users stuck signed in ŌĆö the cookie cache (`SK-AUTH-007`) kept the user authed on the next page load because the cookie clear never ran. CSRF on sign-out is harmless here: `SameSite=Lax` blocks the session cookie from riding cross-site POSTs (no token to delete), and the endpoint only ever clears the caller's own cookie. Bypassing the check costs no real CSRF protection.
- **Consequence in code:** `apps/web/src/pages/auth/sign-out.astro` is the single client sink for every "Sign out" affordance ŌĆö Topnav button, ChatPanel command palette, `ErrorBoundary` fallback, `Base.astro` boot fallback. The page POSTs `/api/auth/sign-out` then `window.location.replace("/")` regardless of outcome, clearing `nlq_hist_*` from localStorage along the way. `apps/web/src/lib/session.ts` exports `fetchSession` only ŌĆö the old `signOut()` helper is gone. PRs that route sign-out back through `auth.handler` or split sign-out across multiple client implementations are rejected at review.
- **Alternatives rejected:** **`advanced.disableOriginCheck: ["/sign-out"]`.** Better Auth's `.d.ts` types `disableOriginCheck` as `boolean | undefined` even though the runtime accepts `string[]`; requires a cast. Direct-API is in-band typed. **`disableCSRFCheck: true` globally.** Disables CSRF on every route ŌĆö silently weakens sign-in and OAuth init. **Custom cookie-clear endpoint without Better Auth.** Skips the `session.delete.after` KV revocation hook (`SK-AUTH-007`) and forces us to mirror cookie naming + attributes in two places. **Keep `auth.handler` + `<a href="/auth/sign-out">` on a server-routed page only.** Doesn't help ŌĆö the GET nav reaches the static page, but the page's POST still 403s through the catch-all.
